Prove real web data, then let the judge call it

Pick a real source — a rocket-launch report, a live price, a match result. Our self-hosted attestor runs a genuine TLS-MPC handshake to it and signs exactly what it served. You watch the proof come back, check it recovers the pinned attestor, and the deterministic judge reads it and settles a yes/no a prediction market would trade — the two launch reports get opposite answers because the judge understands them. Then try to forge the value and watch it break.

source
A real proof takes 30–60 seconds because it is an actual multi-party TLS session, not an HTTP fetch. That is the cost of proving a server really served these bytes. The exact proof you generate here is admissible on-chain (submit_real_zktls.py), verified by native ecrecover.

Prereqs: the attestor (tools/zktls) and the gen server (node tools/reclaim/gen_server.mjs) running locally, plus the judge resolver. See tools/zktls/README.md.
