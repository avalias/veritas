Why this beats the alternatives

Plenty of systems can make an LLM deterministic. The hard part is making a single wrong step cheaply provable on a public chain — without shrinking or distorting the model. Here is the honest landscape.

On that zkML number: the fastest published full-LLM zk systems are minutes-per-token and lossy — zkLLM proves a single 13B forward pass in ~13 min on an A100 while approximating the model; Polyhedra's zkPyTorch runs ~150 s per token on an 8B. A full-fidelity 1B+ judgment proven in seconds is simply not on the table today. Veritas pays zero proving cost on the honest path.

The one idea that makes it work

You do not need to emulate a whole CPU to check a neural network. A network is a few dozen tensor operations, so the on-chain referee only ever has to redo one of them. Four plain ideas carry it:

  • Committed float — the model is pinned to one exact arithmetic recipe (fixed rounding, a fixed addition order), so every CPU and GPU computes the identical answer. Any disagreement is, by definition, one side's provable mistake.
  • Native-speed kernels — there is no CPU emulator in the loop. The honest path runs the real fast kernels (roughly the speed of running the model in ollama); the only extra cost is hashing the work as it goes, so you can prove later.
  • Bisection to one micro-op — instead of re-running ~30 million operations on-chain, challenger and resolver play ~25 rounds of "which half do we disagree on?" until it lands on a single arithmetic step the contract checks for the minimum gas bucket.
  • Arithmetic-agnostic protocol — the commit / bisect / dispute machinery is identical no matter which op is in dispute, so the model stays exact (no quantization) and the contract stays ~330 lines.

As far as our June-2026 survey found, this is the first fraud-proof system to make every micro-operation of a full-fidelity, published-weights LLM adjudicable by a small public contract — and the first on Sui. It opens trustless AI oracles, LLM-resolved prediction markets, and verifiable agents to Sui Move. Honest scope: demonstrated end to end at 0.6B (the protocol is size-agnostic — a bigger model is just more compile time and more genesis pages), and the predictor is currently ~3× slower than llama.cpp on CPU — an engineering gap, not an architectural one.
