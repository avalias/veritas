A new Sui primitive

In the lineage of Sui's trust primitives.

Sui ships native building blocks that turn things you once had to trust into things you can prove: identity, signatures, proofs, attestation. zkLogin took OAuth — a web login — and made it a native, trustless Sui primitive.

opML belongs in that same lineage, for the one verdict no primitive could settle before: did this AI actually run, correctly, on real evidence? It is a foundational building block — potentially a core piece of Sui infrastructure that many teams build on.

Build on opML

You write none of the hard part.

Three steps. The bisection always ends in one place — dispute::verify_step — and the referee is already written.

STEP 01

Compile your model once

opml/compiler turns any quantized LLM (float activations) into a VM program plus genesis pages.

opml/compiler
STEP 02

Run anywhere, commit the root

Run it on CPU (opml/kernels) or Metal (opml/gpu) and commit a Merkle root of the trace — built for free as the forward pass runs.

opml/kernels · opml/gpu
STEP 03

Add the Move package

Drop in the package — now anyone can dispute. Bisection ends in verify_step; integer + float conviction settle on-chain.

opml/move

Authenticated inputs

The judge reads only signed statements.

A fraud proof of garbage is still garbage. So the judge only reads evidence the chain can verify itself — signed by native crypto — with k-of-n independent pinned issuers for safety. No committee, no vote.

The world already signs its facts; opML makes the answer an on-chain object no one has to trust.
