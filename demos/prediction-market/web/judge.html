Watch the AI judge read the news

The real Qwen3-1.7B reads a real news report and settles a market's yes/no question, token by token, live and off-chain. The catch: the report never says "yes" or "no". "Left rates unchanged", "came from behind to beat", "finished second" — the judge has to actually understand it.

Pick a question people would really bet on. Each is paired with a real report whose wording does not match the question, so a keyword search would get it wrong. The judge answers YES, NO, or UNKNOWN with a one-sentence reason. Edit either box, or write your own — it has never seen your line, so it can't be canned.

Why this is the whole product. A prediction market lives or dies on resolving "did X happen?" from messy real-world reports. The Fed didn't say "we did not raise rates" — it said "left unchanged". City didn't say "Arsenal lost" — it said "beat Arsenal 3–2". The judge reads like a person and a market settles on it, with no vote and no committee.

Why one input gives exactly one answer

This model is committed-float: every arithmetic step is pinned to a fixed rounding, so the same report produces the bit-identical output on any CPU or GPU. Ask the exact same thing twice and you get the identical tokens back — normal GPU inference is not even reproducible run to run. Here there is exactly one correct output for a given input.

Determinism is what makes a verdict disputable: a resolver who reports a different output than the one the weights actually produce can be caught and slashed (see slashing.html). The judge only reads the evidence; the market's committed rule decides the outcome, never the AI's opinion.

An honest note on size: this is a 1.7B model, small next to frontier models, so it can still be wrong on a genuinely hard line (and it says UNKNOWN when a report is too vague to call). That is the point, not a bug. The system never trusts the AI to be smart — it makes whatever the AI outputs reproducible and slashable. Swap in a larger model and every guarantee on this page is identical.
