Why one input gives one answer

This model is committed-float: every arithmetic step is pinned to a fixed rounding, so the same evidence produces the bit-identical output on any CPU or GPU. That is unusual — normal GPU inference is not even reproducible run to run. Here there is exactly one correct output for a given input.

Determinism is what makes a verdict disputable: a resolver who reports a different output than the one the weights actually produce can be caught and slashed (see slashing.html). The judge only reads the evidence; the market's committed counting rule decides the outcome, never the AI's opinion.

An honest note on size: this is a 1.7B model, small next to frontier models, so it can still be wrong on a hard line. That is the point, not a bug. The system never trusts the AI to be smart — it makes whatever the AI outputs reproducible and slashable. Swap in a larger model and every guarantee on this page is identical.
